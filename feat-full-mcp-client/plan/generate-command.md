# Plan: cast-mcp-client — `generate` Command

## Goal

Add a `generate` subcommand to `cast-mcp-client` that queries all enabled MCP servers
and for each tool emits a standalone bash script wrapper. The scripts give AI agents
ergonomic, discoverable CLI access to every tool without requiring knowledge of JSON
schemas or the `call` command's invocation syntax.

## Out of Scope

- Generating scripts for stdio/local-process MCP servers (HTTP only, same as parent feature)
- Generating scripts in languages other than bash
- OAuth or dynamic token refresh within generated scripts
- Script manifest file (deferred to Phase 3)

---

## Design Decisions

| Decision | Choice |
|---|---|
| Arg style | Named `--flag value` from `inputSchema.properties` |
| Flag naming | `camelCase` / `snake_case` → `kebab-case` (no regex dep) |
| Arrays / objects | Raw JSON string flag; user passes `'["a","b"]'` or `'{"k":"v"}'` |
| Script runtime dep | `jq` (required in execution environment) |
| Filename convention | Flat: `<server>-<tool-in-kebab>.sh` |
| Tool with no params | Script calls with `'{}'`; only `--help` flag generated |
| Missing `--dir` | Create automatically (`fs::create_dir_all`) |
| Existing scripts | Overwrite silently |
| Non-text MCP content | Print text to stdout; warn on stderr; drop other types |
| MCP `isError: true` | Print text to stderr; exit 1 |
| Transport error | Forward stderr from `cast-mcp-client call`; exit non-zero |
| Executable bit | `chmod 0o755` via `std::os::unix::fs::PermissionsExt` |
| stdout format | JSON: `{ output_dir, scripts: [{server, tool, path}] }` |
| URL in generated scripts | Not hardcoded — delegated to `cast-mcp-client` at runtime via env/config |
| New Rust dependencies | None |

## CLI API

```
cast-mcp-client generate [servers...] --dir <output-dir> [--cast-mcp-url <url>]
```

- `[servers...]` — optional positional varargs; empty = all enabled servers
- `--dir` — required; output directory for generated scripts
- `--cast-mcp-url` — optional; parity with other subcommands (used when fetching tool
  schemas for generation; not embedded in the generated scripts themselves)

## Generated Script Structure

```bash
#!/usr/bin/env bash
# <server>-<tool>: <description>
# Generated by cast-mcp-client generate
# Server: <server> | Tool: <tool>

set -euo pipefail
SERVER="<server>"
TOOL="<tool>"

usage() { cat <<'EOF'
Usage: <server>-<tool> [OPTIONS]
<description>

Options:
  --<flag1> <TYPE>    (required) <description1>
  --<flag2> <TYPE>    (optional) <description2>
  -h, --help          Show this help
EOF
}

# Per-param variables (one per schema property)
<VAR>=""

while [[ $# -gt 0 ]]; do
  case "$1" in
    --<flag>) <VAR>="$2"; shift 2 ;;
    -h|--help) usage; exit 0 ;;
    *) echo "Unknown option: $1" >&2; usage >&2; exit 1 ;;
  esac
done

# Validate required params
[[ -z "${<VAR>:-}" ]] && { echo "Error: --<flag> is required" >&2; exit 1; }

# Build JSON payload via jq
PARAMS=$(jq -n --arg <camelName> "${<VAR>}" '{"<camelName>": $<camelName>}')
# Optional params appended conditionally:
# [[ -n "${OPT_VAR:-}" ]] && PARAMS=$(echo "$PARAMS" | jq --argjson optKey "$OPT_VAR" '. + {"optKey": $optKey}')

# Call and parse
RESULT=$(cast-mcp-client call "$SERVER" "$TOOL" "$PARAMS"); STATUS=$?
[[ $STATUS -ne 0 ]] && { echo "$RESULT" >&2; exit $STATUS; }

IS_ERROR=$(echo "$RESULT" | jq -r '.isError // false')
[[ "$IS_ERROR" == "true" ]] && {
  echo "$RESULT" | jq -r '.content[]|select(.type=="text")|.text' >&2
  exit 1
}

NON_TEXT=$(echo "$RESULT" | jq -r '[.content[]|select(.type!="text")|.type]|unique|join(", ")')
[[ -n "$NON_TEXT" ]] && echo "Warning: ignored non-text type(s): $NON_TEXT" >&2

echo "$RESULT" | jq -r '[.content[]|select(.type=="text")|.text]|join("")'
```

### Type → jq mapping

| JSON Schema type | jq invocation | Notes |
|---|---|---|
| `string` (default) | `--arg name "$VAR"` | Preserves as string |
| `integer` / `number` | `--argjson name "$VAR"` | Preserves numeric type |
| `boolean` | `--argjson name "$VAR"` | User passes literal `true` or `false` |
| `array` / `object` | `--argjson name "$VAR"` | User passes raw JSON string |

### `camel_to_kebab` algorithm (no regex dep)

Walk chars; insert `-` before an uppercase letter that either:
- follows a lowercase/digit, OR
- follows an uppercase and is itself followed by a lowercase

Then lowercase everything and replace `_` with `-`.

Examples: `projectSlug` → `project-slug`, `APIKey` → `api-key`,
`fetch_cast_documentation` → `fetch-cast-documentation`.

## Implementation Phases

### Phase 1 — Core generator [ ]

**Scope:** End-to-end working `generate` command. Single TDD slice. No resilience
against unreachable servers yet (any failure aborts the command).

**New Rust items:**
- `fn camel_to_kebab(s: &str) -> String` — pure, no deps
- `fn generate_script(server_name: &str, tool: &Tool) -> String` — pure string template
- `pub async fn generate_scripts_cmd(filter: Vec<String>, dir: &Path, server_map: HashMap<String, RemoteServerConfig>) -> anyhow::Result<()>`
- `Generate` variant added to `Commands` enum in `main.rs`

**Files changed:** `src/lib.rs`, `src/main.rs`, `tests/mcp_client_test.rs`

**TDD cycles (5 total):**
1. `test_camel_to_kebab` — unit; covers simple camelCase, acronym (APIKey), snake_case, already-kebab
2. `test_generate_script_content` — unit; inspect generated string for correct flags, help text, required markers
3. `test_generate_creates_scripts` — integration; correct filenames, executable bit, JSON output schema
4. `test_generate_script_runs_correctly` — integration; run the script (inject PATH + CAST_MCP_URL), assert stdout
5. `test_generate_script_tool_error` — integration; run script against `error_tool`; assert exit 1 + error text on stderr

**Integration test strategy for running generated scripts:**
The test injects `PATH=<cargo-bin-dir>:$PATH` and `CAST_MCP_URL=http://127.0.0.1:<port>/mcp`
into the `Command` that executes the generated script. The script calls
`cast-mcp-client call cast <tool>` which picks up both from the environment.

**Acceptance criteria:**
- All 5 new tests pass
- All existing 38 tests unaffected
- `cast-mcp-client generate --dir /tmp/tools --cast-mcp-url http://...` produces
  executable scripts that call the correct server/tool

### Phase 2 — Resilience: graceful degradation [x]

Mirror the S7 pattern from `list`: unreachable servers during `generate` emit a
warning to stderr but do not abort generation of scripts for healthy servers.

**Changes to `generate_scripts_cmd`:**
- Wrap each server fetch in `(name, Result<Vec<Tool>>)` tuple (same as `list_tools_cmd`)
- Failed servers → `Warning: server '<name>' is unreachable: <err>` on stderr
- JSON output still emitted (scripts for reachable servers written, others absent)
- Exit code remains 0 when at least one server succeeds

**New TDD cycle:**
- `test_generate_skips_unreachable_server` — two mock servers, one refuses connections;
  verify only healthy server's scripts are written, warning on stderr, exit 0
